Chapter 2: Use Case Diagrams in Requirements Engineering

Use case diagrams are a type of behavioral UML diagram that describe interactions between users and systems to achieve a goal.

Key Concepts

Actor: Any external entity that interacts with the system.

Use Case: A specific functionality or task the system provides to actors.

Include: One use case always triggers another mandatory use case.

Extend: One use case may optionally trigger another use case.

When to Use Use Case Diagrams

Use case diagrams are most valuable during the requirements elicitation phase. They help bridge the gap between technical teams and non-technical stakeholders.

Example:

Actors: Customer, Bank Teller, External Payment Gateway. Use Cases: Login, View Balance, Transfer Funds, Generate Statement.

The level of detail in a use case diagram should match the audience. Always validate diagrams with actual stakeholders before finalizing requirements documentation.
